# Skill：iOS 任务总览笔记

## 我的职责

深度分析 iOS 项目中本次迁移任务涉及的全部代码，
生成一份功能点完整、无遗漏、无死代码的任务总览笔记。

---

## 执行步骤

**Step 1：读取任务信息**

读取 .migration/request.md，明确：
- 任务描述：本次要迁移的页面或功能
- iOS 项目路径
- 用户提供的功能点列表（若有，仅作参考线索，不直接采用）

---

**Step 2：定位任务入口，建立文件清单**

使用 Glob 和 Grep 在 iOS 项目中找到与任务相关的所有入口文件。

寻找策略（多管齐下，不只找主 ViewController）：
- 按任务描述中的关键词搜索文件名和类名
- 搜索 TabBar 配置、路由注册、页面跳转中与任务相关的注册点
- 若用户提供了功能点线索，以线索关键词为起点向外扩散搜索

建立初始文件清单：记录找到的所有入口文件路径。

---

**Step 3：全量 UI 交互扫描**

这是防止遗漏功能点的核心步骤。

对初始文件清单中的每个文件，逐一扫描所有可交互元素：

扫描对象（不能遗漏任何一类）：
- UIButton / SwiftUI Button 的 action / closure
- UIBarButtonItem（导航栏按钮，容易被遗漏）
- UITapGestureRecognizer / UILongPressGestureRecognizer 等手势
- UITableView / UICollectionView 的 didSelectRowAt
- 下拉刷新 UIRefreshControl 的 addTarget
- 上拉加载（scrollViewDidScroll 中的触底判断）
- 滑动手势（左滑删除、右滑操作等）
- 输入框的 delegate 方法（搜索框、筛选输入）
- segmentedControl / tabBar 的切换事件
- 页面生命周期中主动触发的逻辑（viewDidLoad 中的数据加载、viewWillAppear 中的刷新）
- 通知监听（NotificationCenter.addObserver）
- KVO 监听

对每个交互元素记录：
- 所在文件和行号
- 交互类型
- 绑定的处理方法名
- 处理方法是否有实际逻辑（下一步判断）

若扫描中发现新文件（如跳转目标页、被调用的子组件），
将新文件加入文件清单，对新文件同样执行本步骤。
循环直到文件清单不再增长。

---

**Step 4：死代码识别**

对 Step 3 扫描到的每个交互元素，判断其可达性：

**判断规则：**

标记为 DEAD（跳过，不作为功能点）：
- 处理方法存在但方法体为空（`{ }`）或只有注释
- 处理方法存在但在整个项目中零调用（用 Grep 全局搜索方法名确认）
- 被硬编码条件永久屏蔽：
  `if false`、`guard false`、`if (0)`
  `if FeatureFlag.xxx`，且 xxx 在项目中硬编码为 false/0/关闭
  `#if false` 预编译条件

标记为 UNCERTAIN（保留为功能点，在笔记中注明疑虑）：
- 处理方法体只有 TODO / FIXME 注释
- 被 Feature Flag 控制，但 Flag 值来自远程配置或运行时，无法静态判断
- 代码存在但调用链不完整，无法确认是否可达

标记为 ACTIVE（正常作为功能点）：
- 有明确的处理逻辑
- 调用链完整可追溯

**对 DEAD 条目的处理：**
不将其纳入功能点列表，但在笔记末尾的【跳过内容】章节中记录，
说明跳过原因，避免后续阅读者疑惑为什么没有对应功能点。

---

**Step 5：梳理模块结构与调用关系**

基于 Step 3 建立的完整文件清单，梳理：
- 文件目录结构（只列本次任务相关的部分）
- 主要模块划分（页面层 / 组件层 / 网络层 / 数据模型层）
- 模块之间的调用关系和数据流向
- 依赖的第三方库（若有）

---

**Step 6：拆解功能点**

基于 Step 3 的交互扫描结果（仅 ACTIVE + UNCERTAIN 条目）拆解功能点。

**拆解原则：**
- 以用户可感知的完整交互为单位（一个操作路径 = 一个候选功能点）
- 粒度要求：能在一次独立开发会话中完成（不过大也不过小）
- 强耦合的交互可合并为一个功能点（如列表渲染 + 数据加载）
- 复杂功能必须拆分（如一个页面同时包含列表、筛选、详情跳转，拆成多个）
- 按开发依赖顺序排列，被依赖的排在前面
- 基础框架（页面骨架、路由注册、Tab 配置）始终作为第一个功能点

**严格检查（拆解完成后自查）：**
对照 Step 3 的完整交互元素列表，逐一确认每个 ACTIVE/UNCERTAIN 条目
都能在某个功能点中找到对应描述，没有遗漏。

---

**Step 7：写入笔记**

将分析结果写入 .ios-notes/task-overview.md。
